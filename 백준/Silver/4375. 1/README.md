### 문제 설명

<p>2와 5로 나누어 떨어지지 않는 정수 n(1 ≤ n ≤ 10000)가 주어졌을 때, 1로만 이루어진 n의 배수를 찾는 프로그램을 작성하시오.</p>

### 입력 

 <p>입력은 여러 개의 테스트 케이스로 이루어져 있다. 각 테스트 케이스는 한 줄로 이루어져 있고, n이 주어진다.</p>

### 출력 

 <p>1로 이루어진 n의 배수 중 가장 작은 수의 자리수를 출력한다.</p>
 
 
 
 ### 정리
 처음엔 문제가 뭘 말하고자 하는지 이해를 못했다. 조금만 더 친절하게 써주면 좋았을텐데...  1,11,111,1111 이런 규칙을 가진 숫자를 입력 받는 숫자로 나눌때 나머지가 없는 최소의 숫자 자릿수를 출력하는 문제이다. 
* 정리할 부분 1 </br>
입력은 여러 개의 정수를 받는다고 했다. 그래서 반복문을 이용해 정수만 입력 받는걸로 시작했다 

* 정리할 부분 2 </br>
처음 돌렸을때는 시간초과가 떴다 그래서 왜일까 고민했는데 시간초과가 된 코드는 
 ~~~java
 while(sc.hasNextInt()){
             int num = sc.nextInt();
             int n = 0;
             for(int i = 1; ; i++)
             {
                 n = (n * 10) + 1;
                 if(n % num == 0){
                     System.out.println(i);
                     break;
                 }
             }
         }
 ~~~
 이다. 이게 시간 초과가 난 이유는 n이 1,11,111,1111,11111이런식으로 밑도 끝도 없이 커져서 연산과정에서 시간초과가 나는 것 같아서
 ~~~java
  while(sc.hasNextInt()){
             int num = sc.nextInt();
             int n = 0;
             for(int i = 1; ; i++)
             {
                 n = (n * 10) + 1;
                 n = n % num;
                 if(n == 0){
                     System.out.println(i);
                     break;
                 }
             }
         }
 ~~~
 로 효율적인 계산을 위해 나머지 연산(mod)을 이용했다. 구한 값이 int형이나 long형의 표현가능 범위를 벗어나게 되는 경우 mod연산을 통해 크기를 줄일 수 있다.
